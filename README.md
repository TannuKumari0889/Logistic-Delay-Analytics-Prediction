
# 🚚 Logistics Operations Analytics Project

### 📊 Project Overview
This project focuses on analyzing large-scale logistics and transportation data to improve operational efficiency, reduce costs, and enable data-driven decision-making.
The organization operates a complex network involving trucks, drivers, routes, trailers, facilities, and customers. However, the available data was under-utilized and scattered across multiple systems.
This project aims to centralize, analyze, and visualize operational data to provide actionable insights into fleet performance, delivery efficiency, safety, and profitability.

### 🎯 Business Problem
- Lack of end-to-end visibility into logistics operations
- Inefficient tracking of delivery performance and delays
- Rising fuel and maintenance costs without clear root cause analysis
- Inability to consistently evaluate driver and fleet performance
- Reactive reporting instead of proactive decision-making

### 🛠️ Technology Stack
- Excel – Data storage and initial exploration
- SQL / Python – Data cleaning and transformation
- Power BI – Dashboard creation and visualization
- Power BI Service – Report publishing and sharing
- Streamlit – Model deployment and interactive app
- PowerPoint – Stakeholder presentation

### 📂 Dataset Description
The dataset was sourced from Kaggle and consists of 14 interconnected tables, enabling comprehensive logistics analysis:
- Customers
- Delivery Events
- Driver Monthly Metrics
- Drivers
- Facilities
- Fuel Purchases
- Loads
- Safety Incidents
- Trailers
- Trips
- Truck Utilization Metrics
- Trucks
- Routes
- Maintenance Records

### 🔍 Approach & Methodology
##### 1. Data Understanding
- Explored relationships across 14 tables
- Understood operational flow: shipments → trips → deliveries

##### 2. Data Cleaning & Preparation
- Handled minor inconsistencies in data
- Verified absence of duplicates and null values
- Structured data for analysis

##### 3. Exploratory Data Analysis (EDA)
- Analyzed delivery performance and trip patterns
- Studied fuel consumption and maintenance trends
- Identified key operational inefficiencies

##### 4. Data Modeling
- Built relationships between multiple tables
- Created a structured data model for efficient querying

##### 5. Dashboard Development (Power BI)
####### Developed interactive dashboards to monitor:
  - Fleet performance
  - Delivery delays and on-time metrics
  - Fuel and maintenance costs
  - Driver performance
  - Route efficiency

##### 6. Report Publishing
- Published dashboards using Power BI Service
- Enabled easy access and sharing of insights

##### 7. Predictive Modeling
- Built a Trip Delay Prediction Model using Python
- Identified factors contributing to delays

##### 8. Deployment
- Deployed the prediction model using Streamlit App
- Enabled interactive user input for predictions

##### 9. Stakeholder Communication
- Created a structured PowerPoint presentation
- Communicated insights, findings, and recommendations clearly
